Определение стека
Стек – структура данных, работающая по принципу LIFO (Last In – First Out).

В Intel IA-32 стек – область памяти, на которую указывают регистры SS:SP.
Может использоваться для временного хранения значений. Используется при работе с процедурами.
$
Инструкция PUSH
Синтаксис: PUSH src

Принцип работы:
1. SP := SP-2
2. [SS:SP] := src

Назначение:
Помещает значение операнда на вершину стека.
PUSH не изменяет значения флагов.
$
Инструкция POP
Синтаксис: POP dest

Принцип работы:
1. Dest := [SS:SP]
2. SP := SP+2

Назначение:
Извлекает значение с вершины стека.
POP не изменяет значения флагов.
$
Стек
При запуске COM программы в вершине стека находится значения 0.

Правила работы со стеком:
– Все помещенные на стек значения должны быть извлечены;
– Не следует извлекать больше значений, чем было помещено в стек.
$
Команда PUSHF и POPF
PUSHF - помещает значения из регистра флагов на вершину стека.

POPF - перемещает значения с вершины стека в регистр флагов.
$
Команда PUSHA и POPA
PUSHA - cохраняет значения всех регистров общего назначения на стек.

POPA - cчитывает значения всех регистров общего назначения из стека.
$
PUSH SP, POP SP и POP [ESP]
PUSH SP
В Intel 8086 в стек помещается новое значения SP (после уменьшения).

POP SP
Увеличивает значение SP до того, как данные со старой вершины стека будут помещены в SP.

POP [ESP]
Эффективный адрес вычисляется после увеличения ESP.
